## Context

当前粒子系统 (`ParticleField.tsx`) 使用纯径向力场实现鼠标交互：慢速鼠标 (`speed < 2`) 吸引粒子，快速鼠标 (`speed > 5`) 排斥粒子。力场采用线性衰减 `(1 - d/r)` 且通过硬编码的 if/else 分段切换。粒子运动基于 velocity 数组 + 阻尼 (`*= 0.95`)，所有力场通过向 velocity 累加实现。

交互代码集中在 `ParticleField.tsx` 的 useFrame 回调中约 45 行（行 428-474），独立于编队系统、点击效果和光链系统。

## Goals / Non-Goals

**Goals:**
- 用磁场涡旋模型替换现有径向力场，提供切向旋转 + 径向吸引/排斥的复合力
- 旋转方向由鼠标运动方向与粒子位置的叉积动态决定
- 速度-力场关系从硬分段改为 smoothstep 连续过渡
- 力场衰减从线性改为平方反比 (1/d²)
- 增加停留增强效应（dwellTime），鼠标停留越久涡旋越强

**Non-Goals:**
- 不改变编队系统、morph 过渡、呼吸微动
- 不改变点击效果（冲击波、黑洞、长按塌缩）
- 不改变动态光链系统行为
- 不引入新的 shader uniform 或后处理效果
- 不增加粒子的个体属性（如电荷、质量）

## Decisions

### 1. 替换而非叠加现有交互

**选择**: 完全替换现有的径向吸引/排斥代码

**理由**: 磁场涡旋模型天然包含径向分量（吸引/排斥），现有的径向力是磁场力的退化情况（切向分量为零）。叠加会导致力场竞争和参数调优困难。替换后代码更简洁，行为更统一。

**替代方案**: 在现有径向力基础上叠加切向分量 — 拒绝，因为两套力场系统会互相干扰，参数空间变大。

### 2. 叉积决定旋转方向

**选择**: `sign(mouseVel.x * r.y - mouseVel.y * r.x)` 决定切向力方向

**理由**: 这自然产生"搅拌"效果 — 鼠标向右移动时上方粒子顺时针旋转、下方粒子逆时针旋转。与用户的手部运动直觉一致。当鼠标静止时叉积趋近零，切向力自然消失，转为纯径向吸引，避免需要特殊处理。

**替代方案**: 固定旋转方向 — 拒绝，缺乏操控感。

### 3. 连续速度过渡

**选择**: 使用 smoothstep 函数在速度 [1, 6] 范围内连续插值径向力系数从 -1（吸引）到 +1（排斥），切向力系数从 0.5 到 1.0

**理由**: 消除硬阈值处的行为跳变。粒子在鼠标加速/减速时的反应更自然。

### 4. 平方反比衰减 + 最小距离截断

**选择**: 力 ∝ `1 / max(d², minDist²)` 而非 `(1 - d/r)`

**理由**: 平方反比衰减让近处粒子受力更集中，远处衰减更快，产生更有层次的视觉效果。需要最小距离截断 (`minDist² ≈ 0.01`) 防止粒子到达鼠标正下方时力爆炸。同时保留最大半径截断作为性能优化的 early-out。

### 5. dwellTime 停留增强

**选择**: 新增 `dwellTimeRef` 追踪鼠标停留时长。speed < 1 时每帧 `+= delta`，speed > 2 时每帧 `*= 0.9` 快速衰减。力强度乘以 `1 + smoothstep(dwellTime, 0, 4) * 2`，上限 ×3。

**理由**: 给予用户"建造涡旋"的渐进操控感。移开后涡旋通过阻尼自然消散，无需额外清理逻辑。

### 6. 鼠标速度方向追踪

**选择**: 在 useFrame 内通过前后帧鼠标世界坐标差计算速度方向向量，而非修改 store/MouseState

**理由**: 速度方向只在力场计算中使用，不需要暴露给其他模块。保持在 useFrame 本地计算避免不必要的 store 更新和重渲染。使用 ref 存储上一帧鼠标位置即可。

## Risks / Trade-offs

- **[性能]** 平方反比衰减计算量与现有线性衰减相当（都需要 distSq），但取消了 sqrt 的需要（力直接用 distSq），整体可能略快 → 不需要额外优化
- **[手感调优]** 切向力/径向力的比例系数需要反复调试才能达到"有磁场感但不失控"的平衡 → 通过命名常量集中管理，便于快速迭代
- **[边界情况]** 鼠标完全静止时速度方向为零向量，叉积为零，切向力消失 → 这是正确行为，静止时只有径向吸引（如果 dwellTime 增强中），不需要特殊处理
